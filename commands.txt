PostgreSQL replica (первый запуск или после обновления compose):
cd infra
docker compose up -d db
# если том postgres_data уже существовал до репликации:
docker compose exec db sh /scripts/ensure-replication.sh
docker compose restart db
docker compose up -d db-replica

Проверка репликации:
docker compose exec db psql -U cozy_coza -d cozy_coza -c "SELECT pg_is_in_recovery();"
# primary: f

docker compose exec db-replica psql -U cozy_coza -d cozy_coza -c "SELECT pg_is_in_recovery();"
# replica: t

# с хоста: master :5432, replica :5433

Django read/write router (фаза 2):
# web читает с replica, пишет в master; celery и kafka-rating-consumer — только master
docker compose exec web_blue python manage.py shell -c "from django.db import router; from shop.models import Product; print('read:', router.db_for_read(Product)); print('write:', router.db_for_write(Product))"
# read: replica, write: default

docker compose exec web_blue python manage.py shell -c "from django.db import connections; c=connections['replica'].cursor(); c.execute('SELECT pg_is_in_recovery()'); print(c.fetchone())"
# master: заказ и позиции только что записаны, replica может отставать
docker compose exec web_blue python manage.py shell -c "from orders.models import Order; print(Order.objects.using('default').filter(pk=1).db)"
# default
